# 多用户 Web 聊天机器人

## 技术架构

```
浏览器(Angular)  <--WebSocket-->  NestJS后端  <--HTTP Stream-->  Qwen API
                                    |
                              Socket.IO
                           (多用户实时通信)
```

- **前端**: Angular + Socket.IO Client（聊天界面、流式显示）
- **后端**: NestJS + Socket.IO Gateway + OpenAI SDK（调用 Qwen API、管理会话）
- **通信**: WebSocket 实现实时双向通信，支持多用户同时聊天互不干扰

## Task 1: 初始化 NestJS 后端项目

- 使用 `nest new chat-backend` 创建项目
- 安装依赖：`@nestjs/websockets`, `@nestjs/platform-socket.io`, `openai`
- 配置 Qwen API（复用已有的 DashScope Key）

## Task 2: 实现 WebSocket 网关（chat.gateway.ts）

- 创建 `ChatGateway`，监听 WebSocket 连接
- 处理事件：`user_message`（用户发消息）、`stream_chunk`（流式推送给前端）
- 每个用户连接分配唯一 ID，消息按用户隔离

核心逻辑：
```typescript
@SubscribeMessage('user_message')
async handleMessage(client: Socket, payload: { message: string, history: Message[] }) {
  // 1. 调用 Qwen API (stream=True)
  // 2. 逐块通过 WebSocket 推送给该 client
  for await (const chunk of stream) {
    client.emit('stream_chunk', { content: chunk });
  }
  client.emit('stream_done', {});
}
```

## Task 3: 实现 AI 流式调用服务（chat.service.ts）

- 封装 OpenAI SDK 调用，使用 DashScope 兼容接口
- 管理每个用户的对话历史（内存中维护，按 socket ID 隔离）
- 支持流式输出

## Task 4: 初始化 Angular 前端项目

- 使用 `ng new chat-frontend` 创建项目
- 安装依赖：`socket.io-client`
- 创建聊天组件和 WebSocket 服务

## Task 5: 实现聊天界面（chat.component）

- 消息列表区域（区分用户消息和 AI 回复）
- 输入框 + 发送按钮
- 流式接收时逐字追加显示（模拟打字效果）
- 支持 Enter 发送、显示加载状态

## Task 6: 联调测试

- 启动后端 `nest start --watch`
- 启动前端 `ng serve`
- 用两个浏览器标签页模拟多用户同时聊天，验证互不干扰
